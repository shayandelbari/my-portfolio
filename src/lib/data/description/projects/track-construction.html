A web app for managing construction projects, specs, team roles, comments, and file previews. Built with Next.js 15, React 19, and MongoDB.

Features

  • Project Management: Create, edit, and share projects with role-based access (Owner, Editor, Viewer).
  • Specifications: Track specs with tags, due dates, locations, insights, and real-time updates.
  • Collaboration: Comment on specs, upload files (images, PDFs, docs), and preview inline/modal.
  • User Management: Clerk authentication with customizable roles and permissions.
  • UI/UX: Responsive design, dark mode, modals, toasts, and fullscreen support.
  • AI Integration: Optional project/spec generation via Gemini API.

Tech Stack

  • Frontend: Next.js 15 (App Router), React 19, TypeScript, Tailwind CSS, shadcn/ui (Radix-based), Heroicons, Lucide.
  • Backend: Next.js API Routes, MongoDB with Mongoose, Clerk Authentication.
  • State Management: Zustand.
  • Development: ESLint, Prettier, PostCSS.

Getting Started

Prerequisites

  • Node.js 18.x or later
  • MongoDB database
  • Clerk account

Installation

  1. Clone the repo: 
    git clone https://github.com/shayandelbari/track-construction.git
cd track-construction
  2. Install dependencies: 
    pnpm install
  3. Set up environment variables (copy .env.example to .env):
    • MONGODB_URI: MongoDB connection string
    • NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY: Clerk public key
    • CLERK_SECRET_KEY: Clerk secret key
    • GEMINI_API_KEY: Gemini API key (optional for AI features)
  4. Run the dev server: 
    pnpm run dev
  5. Open http://localhost:3000

Project Structure

src/
├── app/                    # Pages and API routes
│   ├── api/               # API endpoints
│   ├── dashboard/         # Dashboard page
│   ├── projects/          # Project pages
│   └── settings/          # Settings page
├── components/            # React components (includes ui/ from shadcn)
├── hooks/                 # Custom hooks
├── lib/                   # Utils, API clients, models
└── types/                 # TypeScript types

API & Authentication

  • API routes under src/app/api/ handle projects, specs, comments, files, permissions.
  • All endpoints require Clerk authentication (include Authorization: Bearer <token> in headers).
  • Roles: Owner (full access), Editor (edit specs/comments/files), Viewer (read-only).
